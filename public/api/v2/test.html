🔔 Firebase Cloud Messaging (FCM) Integration

📱 Push Notifications Setup

TecAudex CRM supports Firebase Cloud Messaging for sending push notifications to mobile apps. Users can register their FCM tokens to receive notifications about tasks, deals, and customer updates.

🔧 FCM Token Management

API Endpoint: PATCH /api/v2/users/{id}/update_fcm_token

Implementation Steps:
1. Get FCM Token: Use Firebase SDK to get registration token
2. Send to Backend: PATCH request with FCM token to update user profile
3. Store Securely: Token is stored in user record for push notifications
4. Auto-Refresh: Update token when it changes (Firebase handles rotation)

Security: Users can only update their own FCM token unless they have admin privileges.

💬 WhatsApp API Integration - Complete Guide

🎯 System Overview

TecAudex CRM integrates with WhatsApp API to provide comprehensive messaging capabilities. The system supports role-based message access, automatic message syncing, and bidirectional communication with customers through WhatsApp Business API.

🏗️ Architecture Components

Backend Components:
• app/services/whatsapp/api_service.rb - WhatsApp API service wrapper
• app/controllers/api/v2/whatsapp_controller.rb - RESTful API endpoints
• app/models/whatsapp_message.rb - Message data model
• app/models/customer.rb - Customer model with WhatsApp integration

Database Schema:
• whatsapp_messages - Stores message history
• customers.whatsapp_chat_id - Links customers to WhatsApp chats
• customers.whatsapp_status - Tracks WhatsApp availability

Background Processing:
• app/workers/whatsapp_sync_worker.rb - Periodic message syncing
• app/workers/whatsapp_analysis_worker.rb - AI message analysis

🔗 Key API Endpoints

GET /api/v2/whatsapp - List all accessible messages
GET /api/v2/whatsapp/customer/{id}/messages - Customer messages
POST /api/v2/whatsapp/customer/{id}/send_text - Send text message
POST /api/v2/whatsapp/customer/{id}/send_image - Send image message
POST /api/v2/whatsapp/customer/{id}/sync - Sync messages from API

🔐 Role-Based Access Control

Admin Access:
• View all WhatsApp messages from all customers
• Send messages to any customer
• Sync messages for any customer
• Full administrative access to WhatsApp features

Manager Access:
• View messages from own customers and assigned associates' customers
• Send messages to customers in their team scope
• Manage WhatsApp communication for their team

Associate/User Access:
• View only their own assigned customer messages
• Send messages to their assigned customers only
• Limited to their customer portfolio

⚙️ Configuration Setup

Required Credentials (Rails.application.credentials):
• WAAPI_INSTANCE_ID - Your WhatsApp API instance ID
• WAAPI_AUTH_TOKEN - Your WhatsApp API authentication token

WhatsApp API Provider:
• Base URL: https://waapi.app/api/v1/instances/{instance_id}
• Authentication: Bearer token in Authorization header
• Supported features: Text, media, chat management

Database Configuration:
• WhatsApp chat IDs automatically synced with customer phone numbers
• Message history stored with timestamps and metadata
• Customer-message relationships maintained

⚡ Real-time Features

Auto-Polling:
• 5-second interval message polling in test interface
• Real-time message updates with timestamps
• Automatic authentication error handling

Message Synchronization:
• Bi-directional message sync with WhatsApp API
• Duplicate message prevention
• Batch message fetching for performance

Chat ID Management:
• Automatic chat ID resolution from phone numbers
• WhatsApp registration verification
• Persistent chat ID storage for customers

🔒 Security & Permissions

Access Control:
• JWT-based authentication required for all endpoints
• Pundit authorization policies enforce role-based access
• Users can only access their permitted customer scope
• API rate limiting and request validation

Security Measures:
• Secure credential storage in Rails credentials
• HTTPS-only API communication
• Input sanitization and validation
• Error message sanitization to prevent information leakage

📞 Twilio Calling System - Complete Guide

🎯 System Overview

TecAudex CRM integrates with Twilio to provide browser-based calling, automatic call recording, and call management features. The system supports both outbound calling from the web interface and inbound call routing to sales representatives.

🏗️ Architecture Components

Frontend Components:
• app/javascript/twilio_client.js - Main Twilio client implementation
• app/javascript/controllers/calling_controller.js - Stimulus controller for UI
• app/views/calling/index.html.erb - Calling interface view

Backend Components:
• app/controllers/calling_controller.rb - Handles calling endpoints
• app/services/twilio_service.rb - Twilio API service wrapper
• config/initializers/twilio.rb - Twilio configuration
• app/models/recording.rb - Call recording data model

Background Processing:
• app/workers/recording_storage_worker.rb - Handles recording storage to S3
• app/workers/customer_phone_analysis_worker.rb - AI analysis of calls

⚙️ Configuration Setup

Required Credentials (Rails.application.credentials):
• TWILIO_ACCOUNT_SID - Your Twilio Account SID
• TWILIO_AUTH_TOKEN - Your Twilio Auth Token
• TWILIO_APP_SID - TwiML Application SID

TwiML App Configuration:
• Voice URL: https://crm.tecaudex.com/calling/voice
• Voice Method: POST
• Status Callback URL: https://crm.tecaudex.com/calling/recording_status
• Status Callback Method: POST

Default Settings:
• Default Caller ID: +447897021964
• Recording Format: MP3
• Recording Storage: AWS S3

🔄 Call Flow Process

Outbound Call Flow:
1. User selects customer and clicks Call in browser interface
2. Frontend requests capability token from /twilio/token
3. Twilio Device initializes with token and connects to Twilio
4. Call parameters sent to TwiML app at /calling/voice
5. TwiML response dials target number with recording enabled
6. Call recording status posted to /calling/recording_status
7. Recording stored to S3 and customer activity logged

Inbound Call Flow:
1. Incoming call triggers TwiML app webhook
2. System identifies caller and assigned sales rep
3. Call forwarded to sales rep's phone with recording
4. Recording processed and stored same as outbound calls

🔗 Key API Endpoints

GET /calling - Main calling interface
GET /twilio/token - Generate capability token
POST /calling/voice - TwiML voice webhook
POST /calling/recording_status - Recording callback
GET /twilio/recordings - Fetch call recordings
GET /calling/available_numbers - Get Twilio numbers
POST /calling/store_customer_id - Associate call with customer

🔒 Security & Permissions

Security Measures:

🔧 Troubleshooting Guide

Common Issues & Solutions:

1. Device not ready / Token errors:
• Check Twilio credentials in Rails.application.credentials
• Verify TwiML app configuration in Twilio Console
• Ensure capability token generation is working

2. Microphone access denied:
• Browser must be served over HTTPS
• User must grant microphone permissions
• Check browser compatibility (Chrome/Firefox recommended)

3. Call connection fails:
• Verify phone number format (+country code)
• Check Twilio account balance
• Ensure caller ID is verified in Twilio

4. Recordings not saving:
• Check webhook URL accessibility
• Verify S3 credentials and permissions
• Monitor Sidekiq job queue for failed workers
